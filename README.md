# PredictX - Decentralized Prediction Market

PredictX is a full-stack decentralized prediction market where users can bet on the outcomes of future events (e.g., Crypto, Politics, Sports, Tech) using USDC. The platform is entirely on-chain, utilizing smart contracts for pool management, parimutuel betting, and outcome resolution.

## Architecture

The project is structured into three main components:

- **`/contracts`**: The blockchain layer (Solidity + Hardhat). Contains the smart contracts for the Market Factory, individual Prediction Markets, an Oracle service, and a Mock USDC token for testing.
- **`/backend`**: An Express.js Node API that listens to blockchain events (like `MarketCreated`). It tracks active markets, handles automated syncing, and serves real-time market data to the frontend so it doesn't have to query the blockchain for everything constantly.
- **`/frontend`**: A React + Vite frontend styled with Tailwind CSS. It connects to MetaMask, allows users to buy "YES" or "NO" shares with their USDC, creates new markets on-chain, and claims winnings once a market resolves.

## Prerequisites

- **Node.js** (v18 or newer recommended)
- **MetaMask** browser extension

## Getting Started (Local Development)

To run the entire stack locally, follow these steps:

### 1. Start the Blockchain & Deploy Contracts

Open a terminal and navigate to the `contracts` directory:

```bash
cd contracts
npm install
# Start the local Hardhat node (Leave this terminal running!)
npx hardhat node
```

Open a **second terminal** in the `contracts` directory to deploy the contracts:

```bash
cd contracts
npx hardhat run scripts/deploy.ts --network localhost
```

This will deploy the MarketFactory, MockUSDC, SimpleOracle, and three demo prediction markets. It will automatically write the new contract addresses to a `deployed-addresses.json` file in both the `frontend` and `backend` directories. It will also mint 10,000 test USDC to the first few Hardhat accounts.

### 2. Configure MetaMask

1. Open MetaMask and add a custom network:
   - **Network Name**: Hardhat Localhost
   - **RPC URL**: `http://127.0.0.1:8545`
   - **Chain ID**: `31337`
   - **Currency Symbol**: `ETH`
2. Import one of the test accounts generated by the Hardhat node (Accounts #1 through #4 have the minted USDC) using the private keys printed in the terminal.

### 3. Start the Backend

Open a **third terminal** and navigate to the `backend` directory:

```bash
cd backend
npm install
npm run dev
```

*(Note: In a production environment, you would add an `.env` file here with `ORACLE_PRIVATE_KEY` and an `API_KEY` to allow the backend oracle service to resolve markets automatically).*

### 4. Start the Frontend

Open a **fourth terminal** and navigate to the `frontend` directory:

```bash
cd frontend
npm install
npm run dev
```

Open `http://localhost:8080` in your browser. Ensure your MetaMask is connected to the Localhost network, and you're ready to trade!

## How It Works

1. **Market Creation**: Any user can create a market by supplying an expiry date and a question. The `MarketFactory` deploys a new `PredictionMarket` contract.
2. **Trading**: Users approve the market contract to spend their USDC and call `buyYes` or `buyNo`. The money goes into a global pool for that specific market.
3. **Resolution**: After the expiry date, an Oracle (currently a trusted account) submits the final outcome (`YES` or `NO`).
4. **Claiming**: If a user bet on the winning side, they can claim their proportional share of the entire pool (minus any protocol fees, if implemented).
